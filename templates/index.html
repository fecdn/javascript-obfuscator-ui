JavaScript 混淆工具

一个免费、高效的 JavaScript（包含 ES6）的混淆器。让您的代码变得难以复制、阻止他人盗用您的作品。

这是什么？

这个工具将您的原始 JavaScript 源码转义为一种新的表达，更加难以阅读、复制、复用及未经授权修改。混淆结果将会保持原始代码的确切功能。

另外，请先阅读参数选项的相关说明，理解代码保护和代码大小/速度之间的所有权衡。

与UglifyJS, Closure Compiler相似吗？

是也不是。UglifyJS（和其他压缩工具）让输出的代码更难以理解（压缩、丑化），但依然可以使用 JS 美化工具轻松转成方便阅读的代码。

T该工具通过使用各种变换和“陷阱”来防止这种情况，例如自我防御和调试保护。

如何混淆的？

通过一系列变换，例如变量/函数/参数重命名，字符串删除和其他操作，您的源代码被转换为不可读的东西，同时能够像以前一样工作。

锁定混淆的源代码，使其只能在特定域和/或子域上运行。这使得真的很难为某人只是复制和粘贴您的源代码和运行在其他地方。

Source Map

确保不要上传嵌入了Source Map的混淆源代码，因为它包含您的原始源代码。

源映射可以帮助您调试模糊的Java Script源代码。如果您想要或需要在生产中调试，您可以将单独的源映射文件上传到一个秘密位置，然后将浏览器指向那里。在Google Chrome开发人员工具网站上详细了解Source Map。

内联源映射（Inline Source Map）

这会将源的源映射嵌入到模糊代码的结果中。如果你只想在本地调试机器上有用。

单独源映射(Separate Source Map)

这将使用源映射生成单独的文件。用于在生产中调试代码，因为这使您能够将源映射上传到服务器上的秘密位置，然后指向浏览器使用它。

使用Source Map Base URL 和 Source Map 文件名 来自定义 sourceMappingURL 将被附加到模糊代码末尾的属性。

例如，如果你设置 Base URL 为 "http://localhost:9000" 和 文件名 为 "example"，你会得到： //# sourceMappingURL=http://localhost:9000/example.js.map。附加到加密代码的末尾。

常见问题 FAQ

为什么要加密JavaScript代码？

保护您的代码是一个好主意，这样做有众多原因，例如：

  • 防止任何人简单地复制/粘贴您的工作。这对于100％客户端项目（如HTML5游戏）尤其重要;
  • 删除不需要的注释和空格。使加载更快，更难理解;
  • 保护尚未支付的工作。您可以向客户显示您的工作，因为他们知道他们将不会有源代码，直到发票已支付。

这个混淆器绝对万无一失？

不，虽然不可能恢复确切的原始源代码，有时间，知识和耐心的人可以逆向工程。

由于JavaScript在浏览器上运行，浏览器的JavaScript引擎必须能够读取和解释它，因此没有办法阻止。任何承诺的工具不是诚实的。

为什么我的混淆代码大于我的原始来源？

因为混淆器引入了旨在保护和防御调试和逆向工程的新代码。另外，字符串被转换为\xAB十六进制代码，使事情有点难以理解。你不必担心代码大小，因为有很多重复，被混淆的代码将被你的网络服务器压缩得非常好（如果你的服务器上启用了GZIP压缩，现在大多数）。

我可以在模糊输出上运行一个minifier如UglifyJS或Google Closure Compiler？

不，不推荐，在某些情况下，它会破坏代码（例如，如果你启用了自我保护）。你可以运行你的代码通过minifier之前，以确保它删除死代码和做其他优化。

你存储我的源代码吗？

不。源代码由我们的应用程序服务器处理，然后到混淆器并返回浏览器，所以它只在我们的服务器内存上保留一段短暂的时间（通常是毫秒）。

我可以从混淆的恢复原始的源代码吗？

不，不可能将模糊代码还原为原始代码，因此请保持原始代码的安全。

此工具是否可以与Node.js源代码一起使用？

可以。
